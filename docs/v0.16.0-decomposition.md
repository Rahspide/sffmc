# SFFMC v0.16.0: God-class decomposition porting guide

## Overview

v0.16.0 decomposed 5 god-class files into 22 focused sub-modules. Public API of
each package is unchanged — every package's `index.ts` still re-exports the
same names with the same behavior. This guide helps you find where specific
logic now lives if you imported from internal files directly.

## Mapping table

| Old (v0.15.x) | New (v0.16.0) |
|---|---|
| `packages/memory/src/extra/dream.ts` (1291 LOC) | `packages/memory/src/extra/dream.ts` (barrel, 10 LOC) — re-exports from `dream-types.ts`, `dream-db.ts`, `dream-dedup.ts`, `dream-clustering.ts`, `dream-llm.ts`, `dream-orchestrator.ts` |
| `packages/runtime/src/runtime.ts` (817 LOC) | `packages/runtime/src/runtime.ts` (614 LOC) + `script-launcher.ts`, `recovery.ts` |
| `packages/memory/src/extra/judge.ts` (657 LOC) | `packages/memory/src/extra/judge.ts` (barrel, 10 LOC) — re-exports from `judge-types.ts`, `judge-prompt.ts`, `judge-parse.ts`, `judge-llm.ts`, `judge-extract.ts`, `judge-tool.ts` |
| `packages/runtime/src/mcp.ts` (335 LOC) | `packages/runtime/src/mcp.ts` (barrel, 26 LOC) + `mcp-types.ts`, `mcp-resolver.ts`, `mcp-bridge.ts` |
| `packages/cognition/src/max-mode/src/index.ts` (328 LOC) | `packages/cognition/src/max-mode/src/index.ts` (barrel, 31 LOC) + `max-mode-config.ts`, `max-mode-winner.ts`, `max-mode-hooks.ts` |
| `packages/runtime/src/constants.ts` (345 LOC) | `packages/runtime/src/constants.ts` (barrel, 17 LOC) + `constants-defaults.ts`, `constants-config.ts` |

## Public API

Each package's `index.ts` re-exports the public API. If you imported from a
god-class file directly, you should now import from the package's public
`index.ts`.

Before:

```ts
import { MAX_AGENT_INSTRUCTIONS } from "@sffmc/runtime/src/constants";
```

After:

```ts
import { MAX_AGENT_INSTRUCTIONS } from "@sffmc/runtime"; // re-exported from public index
```

## Per-package changelog

### @sffmc/memory

- `extra/dream.ts` is now a barrel (10 LOC). Same exports, same behavior. All
  callers should already work via the public `index.ts`.
- `extra/judge.ts` is now a barrel (10 LOC). Same.

If you reached into `dream-clustering.ts`, `judge-parse.ts`, or similar internal
helpers, update to the new paths — the old path is now a barrel that only
re-exports the top-level public API.

### @sffmc/runtime

- `runtime.ts` shrank from 817 → 614 LOC. The `WorkflowRuntime` class is still
  public via `index.ts`.
- `mcp.ts` shrank from 335 → 26 LOC. `McpBridge` is re-exported.
- `constants.ts` shrank from 345 → 17 LOC. All constants are re-exported.

The split:

- `script-launcher.ts` — process spawn / IPC plumbing extracted from
  `runtime.ts`.
- `recovery.ts` — restart-and-reconnect logic extracted from `runtime.ts`.
- `mcp-types.ts` — MCP type definitions.
- `mcp-resolver.ts` — server resolution / lookup.
- `mcp-bridge.ts` — actual bridge implementation.
- `constants-defaults.ts` — hardcoded default values.
- `constants-config.ts` — config-overridable constants.

### @sffmc/cognition

- `max-mode/src/index.ts` shrank from 328 → 31 LOC. Config, winner picker, and
  hooks live in three sub-modules: `max-mode-config.ts`, `max-mode-winner.ts`,
  `max-mode-hooks.ts`.

## Migration

- **No action needed** if you import from the package root (`@sffmc/<pkg>`).
- If you imported internal helpers from `extra/dream-clustering.ts` etc.,
  update to the new paths.
- Run your tests after upgrading — the public API is preserved, but if you
  reached into private internals, behavior may have shifted.
